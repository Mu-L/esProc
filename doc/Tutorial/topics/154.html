5.5.2 Cross-cellset call

Besides the subroutine call, esProc also supports the cross-cellset call, which uses call function to run a program from another cellset or to make use of the result of another cellset. This means a whole cellset will be treated as a subroutine to be executed. The return statement is used to return the result. End the cross-cellset call to release the memory after the result is returned. D:\files\createID.splx is a cellset:

 

A

B

1

/create an ID

 

2

 

>size.run(A2=A2+char(65+rand(26)))

3

return A2

 

In the cellset, B2 adds characters to A2 circularly to create random IDs and returns them using return statement. Similar to a subroutine call, it is also not necessary to use the return statement for a cross-cellset call. The returned result will be the value of the last calculation cell starting with =. You can change A3¡¯s code to =A2, for example, and still get the same result. Notice that B2 uses a cellset parameter, size, which needs to be set in Program parameter dialog on the menu bar.

If a parameter is needed during the cross-cellset call, it should be transferred through the cellset parameter used in the splx file being called. The way of using call function to call a program in another cellset is similar to that of calling a subroutine:

 

A

1

=call("D:\\files\\createID.splx",rand(3)+3)

2

=call("createID.splx",rand(3)+3)

Likewise, the parameter is put after the program being called. Difference is that the script file being called is specified directly through its name during the cross-cellset call in A2. Note that \ is an escape character when used in a string expression; so, in A1¡¯s code another \ is put before it to represent the character literally. By reusing the existing cellset program, the cross-cellset call makes the code in the main cellset more concise. In addition, an algorithm which needs to be encrypted can be put into a separate cellset for being called by another program.

When using parameters in a cellset file, we can set the last parameter as a dynamic parameter. During cellset execution, values for the last parameter will be concatenated as a sequence to be passed to the cellset file, like the following file D:\files\findEmployee.splx shows:

 

A

1

/find employees in given states

2

=connect("demo")

3

=A2.query@x("select EID, NAME, SURNAME, STATE, GENDER, DEPT from EMPLOYEE where DEPT=? and STATE in (?)", dept, states)

4

return A3

The cellset file searches for employee records of the specified departments and states in the database. To connect to the data source, we need to configure datasoruce-related connection parameters in configuration file raqsoftConfig.xml.

If a cellset file needs to be repeatedly called, use register(func, spl) to register script spl as function func. Then we can use the registered function name func to directly call the script executing the cellset file, for example:

 

A

1

=register("findEmps", "findEmployees.splx")

2

=findEmps("Finance", "Texas", "New York", "Florida")

3

=findEmps("R&D", "Pennsylvania", "North Carolina")

According to A2¡¯s result, after register the script as a function, the result of calling it is the same as that of using call() function to execute the cellset file.

A registered function is always invalid unless esProc is exited. For example, it can be executed in a newly-created main cellset:

 

A

1

=findEmps("Finance", "Texas")

2

And gets the query result correctly.
